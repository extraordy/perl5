Properties - Quiz 1

Given $_ = "one, two, three, four... go!", you want to match the last word before a comma and come up with the regular expression /.*(\w+),/ but that doesn't work. What does it match instead, and why?

It matches the last "e" in "three." This is because the .* is greedy and matches as far as it can, backtracking as necessary, until there is just enough to satisfy the rest of the regex, and the subpattern \w+ only needs one character to match.

Grading Criteria: Student should demonstrate understanding of how a regular expression will backtrack as needed.

The lvalue ternary operator is often abused. You encounter a program containing the code:

Observe:
($var eq 'Ford' ? $ford : $var eq 'Chrysler' ? $chrysler : $var eq 'Buick' ? $buick : $import) += 1

What data type would be a far better representation for this operation?

A hash should be used here.

Some students will point out, though, that this is not a direct replacement as a hash does not have a default fail over ($import).

Grading Criteria: Student should demonstrate knowledge of data types as possible replacements for logical code structures.
