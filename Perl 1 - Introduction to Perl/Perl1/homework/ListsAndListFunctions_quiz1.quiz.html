Properties - Quiz 1

John Q. Programmer wanted to print out a line with three strings separated by asterisks, and so he wrote:

Observe:
print join '*', 'a', 'b', 'c', "\n";

What's wrong with that code and how would you fix it?

The reason this is an "error" is that the developer probably did not want a*b*c*\n; rather they wanted a*b*c\n (without the extra * between c and \n).

Observe:
print join('*', 'a', 'b', 'c'), "\n";

If the student has split up the list by saving off $string as the join of a,b,c and then prints "$string\n", inform them that while this does address the issue there is a simpler way that only requires the use of two additional characters in the example code.

Note that this will not work: print (join '*', 'a', 'b', 'c'), "\n";

Grading Criteria: Student should demonstrate knowledge of the join statement.

In the statement:

Observe:
($x, $y, $z) = (1, 2);

What does $z get set to?

Observe:
undef

Other common answers that define the same concept are: Uninitialized, Null, and Nil. If these are given, tell the student that they are correct but in Perl we call this concept undef.

Grading Criteria: Student should demonstrate understanding of how an empty value in a list is handled.
