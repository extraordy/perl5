Scalars and Arithmetic - Quiz 2

Create a new perl script with the following code:

Type this in the editor below:
 my $x = (.0425 * 2) + .0025;
 print $x - (.0425 * 2) - .0025;

Run your script. What answer do you get?

Explain why the answer is not zero.

2.16840434497101e-18 due to floating point precision rounding errors.

Note: There is some possibility that the answer will change depending on the platform that the code is run on, but this is the answer that the student will get when they run the program on our platform.

The reason is due to rounding errors and the limited precision of floating point numbers as stored by computers. The point of this question is not for the student to understand exactly how this happens and the structure of floating point storage and calculations, but to understand that it does happen, because it is a source of frequent frustration in Perl and all languages. Any answer that includes the terms "rounding" and "precision" is likely to be correct.

An answer that refers to improper use of () or precedence may indicate the student has altered the given code. This answer should be considered incorrect; ask the student to run the code as is and explain why the answer that it does result in is not zero, and instead is a very very small number that approaches zero.

Grading Criteria: Student should show they recognize floating point errors.
